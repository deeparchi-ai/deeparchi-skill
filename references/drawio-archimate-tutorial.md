# Draw.io ArchiMate 教程

## 快速开始

### 步骤 1：打开 Draw.io

1. 访问 https://app.diagrams.net/
2. 点击 "Create New Diagram"
3. 选择 "Blank Diagram"

### 步骤 2：启用 ArchiMate 形状库

1. 在左侧面板点击 "More Shapes"（更多形状）
2. 在搜索框中输入 "ArchiMate"
3. 勾选 "ArchiMate 3.2" 或相关形状库
4. 点击 "Apply"

### 步骤 3：开始绘制

现在左侧面板会显示 ArchiMate 形状库，包含：
- Business Layer（业务层）
- Application Layer（应用层）
- Technology Layer（技术层）
- Motivation & Strategy（动机与策略）
- Implementation & Migration（实施与迁移）

## 创建业务架构视图

### 示例：银行开户业务流程

1. **添加业务角色**
   - 从 Business Layer 拖拽 "Business Role"
   - 命名为 "客户"
   - 再添加 "银行柜员"、"风控部门"

2. **添加业务流程**
   - 拖拽 "Business Process"
   - 命名为 "开户流程"
   - 设置颜色为业务层标准色（黄色系）

3. **添加业务服务**
   - 拖拽 "Business Service"
   - 命名为 "账户管理服务"
   - 命名为 "身份验证服务"

4. **建立关系**
   - 使用 "Assignment" 关系连接业务角色和业务流程
   - 使用 "Realization" 关系连接业务流程和业务服务
   - 使用 "Serving" 关系连接业务服务和业务角色

## 创建应用架构视图

### 示例：支付系统应用架构

1. **添加应用组件**
   - 从 Application Layer 拖拽 "Application Component"
   - 命名为 "核心系统"
   - 命名为 "支付系统"
   - 命名为 "风控系统"

2. **添加应用接口**
   - 拖拽 "Application Interface"
   - 命名为 "REST API"
   - 命名为 "消息队列接口"

3. **添加数据对象**
   - 拖拽 "Data Object"
   - 命名为 "账户数据"
   - 命名为 "交易数据"

4. **建立关系**
   - 使用 "Serving" 关系连接应用组件
   - 使用 "Access" 关系连接应用组件和数据对象
   - 使用 "Realization" 关系连接应用接口和应用组件

## 创建技术架构视图

### 示例：系统部署架构

1. **添加技术节点**
   - 从 Technology Layer 拖拽 "Node"
   - 命名为 "应用服务器"
   - 命名为 "数据库服务器"

2. **添加系统软件**
   - 拖拽 "System Software"
   - 命名为 "操作系统"
   - 命名为 "数据库管理系统"
   - 命名为 "中间件"

3. **添加网络**
   - 拖拽 "Network"
   - 命名为 "内部网络"
   - 命名为 "外部网络"

4. **建立关系**
   - 使用 "Assignment" 关系连接系统软件和技术节点
   - 使用 "Communication Path" 连接不同节点
   - 使用 "Realization" 关系连接系统软件和技术服务

## 样式和格式

### 颜色规范

ArchiMate 标准颜色：
- **业务层**：#FFF2CC（浅黄）、#FFD966（深黄）
- **应用层**：#D5E8D4（浅绿）、#97D077（深绿）
- **技术层**：#D5E8D4（浅绿）、#97D077（深绿）

### 设置元素颜色

1. 选中元素
2. 在右侧面板找到 "Fill" 选项
3. 选择标准颜色或输入颜色代码

### 文本格式

- 使用清晰的字体（如 Arial、微软雅黑）
- 字号建议：标题 14pt，正文 12pt
- 使用粗体突出重要元素

## 关系连接

### 添加关系

1. 点击工具栏的 "Connector" 工具
2. 从一个元素拖拽到另一个元素
3. 选择合适的关系类型

### 关系样式

- **实线**：结构关系（组合、聚合、分配、服务、关联、流）
- **虚线**：动态关系（实现、访问、触发、影响）

### 关系方向

- 确保箭头指向正确的方向
- 在右侧面板可以调整箭头样式

## 布局技巧

### 对齐和分布

1. 选中多个元素
2. 使用工具栏的对齐工具：
   - 左对齐、右对齐、居中对齐
   - 顶部对齐、底部对齐、垂直居中
   - 水平分布、垂直分布

### 网格和对齐线

1. 启用网格：View → Grid
2. 启用对齐线：View → Guides
3. 启用对齐：View → Snap to Grid

### 层次布局

- 使用分层布局展示架构层次
- 业务层在上，应用层在中，技术层在下
- 使用容器或分组组织相关元素

## 导出和分享

### 导出格式

1. File → Export as
2. 选择格式：
   - PNG（高质量图片）
   - SVG（矢量图）
   - PDF（文档格式）
   - XML（draw.io 原生格式）

### 导出设置

- **分辨率**：建议 300 DPI 用于打印
- **背景**：选择透明或白色背景
- **边框**：添加或移除边框

### 分享图表

1. File → Share
2. 生成分享链接
3. 或导出后通过其他方式分享

## 高级技巧

### 使用模板

1. File → New from Template
2. 选择 ArchiMate 模板
3. 基于模板快速创建图表

### 自定义形状

1. 创建自定义形状库
2. 保存常用元素组合
3. 重复使用提高效率

### 图层管理

1. 使用图层组织复杂图表
2. 可以隐藏/显示特定图层
3. 便于管理和编辑

## 常见问题

### Q: 找不到 ArchiMate 形状库？

A: 确保在 "More Shapes" 中启用了 ArchiMate 3.2 形状库。如果仍然找不到，可能需要更新 draw.io 或使用在线版本。

### Q: 如何添加关系标签？

A: 双击关系线，输入标签文本。可以在右侧面板调整标签位置和样式。

### Q: 如何批量修改元素样式？

A: 选中多个元素，在右侧面板统一修改样式属性。

### Q: 如何创建跨层视图？

A: 在同一图表中使用不同层的元素，使用跨层关系（如实现关系）连接它们。

## 最佳实践

1. **保持简洁**：避免在单个视图中包含过多元素
2. **使用标准元素**：遵循 ArchiMate 标准，避免自定义形状
3. **关系清晰**：确保关系类型使用正确，方向明确
4. **标注完整**：为图表添加标题、图例和说明
5. **版本管理**：使用版本号管理图表版本
6. **定期更新**：保持图表与实际情况同步
